# Contributing to EcoScript

Thanks for being interested in contributing! This document explains the preferred workflow and repository conventions.

- Fork the repository and create a topic branch for your changes.
- Keep changes small and focused. Open a draft PR early for discussion.
- Ensure unit tests pass and add tests for new features or bug fixes.
- Follow existing style and add documentation where helpful.

If you're unsure how to begin, open an issue describing what you'd like to work on and we'll help get you started.
